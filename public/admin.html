Claude Desktop Redirect

Run once in Administrator PowerShell — tells Claude Desktop to route through this proxy:

[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_URL", "http://127.0.0.1:8082", "Machine")
Revert to Real Anthropic

Remove the redirect and restart Claude Desktop:

[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_URL", $null, "Machine")

Or switch to the Real Claude (Anthropic) provider in the Providers tab — no env var change needed.

Help
Quick reference.
How it works
Claude Desktop ↓ http://127.0.0.1:8082 (this proxy) ↓ Active Provider → NVIDIA / Groq / Ollama / Real Claude / ...
Daily usage
Start the proxy: .\start.ps1 or install the auto-start service once with .\install-service.ps1.
Then open Claude Desktop normally. Switch providers anytime here — no restart needed.
Validate which model is running
Check the status bar at the top of this page, or watch the proxy console window — each request prints the provider and model. The Request Log tab also shows full history with token counts and latency per request.
Adding API keys
Click Edit on any provider → paste key in the API Key field → click Save Key. Takes effect immediately on the next request. Key is stored in config.json and never sent to the browser in full — only the last 4 characters are shown.
